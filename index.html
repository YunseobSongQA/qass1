테스트에 돌려드립니다

QASS는 웹 서비스 QA 과정에서 가장 번거로운 일 — 화면 캡처, 파일 정리, 결함 기록 — 을 자동화하는 무료 증적 관리 플랫폼입니다. 탭을 이동하기만 해도 풀페이지 캡처가 팀 공유 공간에 저장되고, 맞춤법·띄어쓰기·UI 장애까지 자동으로 점검해 분류합니다.

주요 기능

📸

자동 풀페이지 캡처

크롬 확장 프로그램의 시작 버튼을 누르면, 이후 탭을 전환하거나 페이지를 스크롤할 때마다 화면 전체가 자동으로 캡처됩니다. 테스트하면서 일일이 캡처 도구를 켤 필요가 없습니다. 페이지 안쪽 스크롤 영역까지 스캔해 긴 페이지도 끊김 없이 한 장으로 저장됩니다.

📁

방(Room) 기반 팀 공유

프로젝트나 테스트 회차별로 방을 만들고 비밀번호를 팀원과 공유하면, 모든 팀원의 캡처가 한 공간에 실시간으로 모입니다. 업로더 이름과 캡처 시각이 자동 기록되고, URL·제목 검색과 업로더 필터, ZIP 일괄 다운로드를 지원합니다.

🔍

1차 자동 점검

캡처하는 순간 페이지의 텍스트와 화면 요소를 룰 기반으로 검사해 맞춤법 오류, 띄어쓰기 오류, 텍스트 잘림 같은 UI 장애를 찾아냅니다. 검출된 위치는 원본 이미지를 훼손하지 않는 오버레이 박스로 표시되어, 검증 전·후를 나란히 비교할 수 있습니다.

📱

모바일 대응 대시보드

대시보드는 PC와 모바일 어디서든 동일하게 동작합니다. 외부에서 급히 증적을 확인해야 할 때 휴대폰으로 접속해 캡처를 열람·다운로드할 수 있고, 모바일에서 직접 촬영한 이미지를 업로드해 증적으로 추가할 수도 있습니다.

작동 방식

1

방 만들기

대시보드 또는 확장 프로그램에서 방을 만들고 비밀번호를 팀원과 공유합니다.

2

캡처 시작

확장 프로그램에서 방에 연결하고 시작을 누르면, 테스트 중 화면이 자동 캡처·업로드됩니다.

3

점검 결과 확인

대시보드의 1차 자동 점검 탭에서 맞춤법·띄어쓰기·UI 장애를 한눈에 확인합니다.

QA 증적 관리, 왜 중요할까요?

소프트웨어 검수에서 "증적(Evidence)"은 테스트를 실제로 수행했고 그 시점에 화면이 어떤 상태였는지를 입증하는 기록입니다. 발주처 검수, 오픈 전 최종 점검, 장애 원인 분석 등 실무의 결정적인 순간마다 증적의 유무가 결과를 좌우합니다. "그때는 정상이었다"는 말은 증적이 없으면 아무런 힘이 없습니다.

문제는 증적 수집이 대표적인 비생산 업무라는 점입니다. 테스터는 화면을 확인할 때마다 캡처 도구를 켜고, 파일 이름을 규칙에 맞게 바꾸고, 폴더를 나누어 정리하고, 누락이 없는지 다시 확인해야 합니다. 수백 페이지 규모의 프로젝트라면 이 작업에만 전체 QA 시간의 상당 부분이 소모되고, 사람이 하는 일이라 누락과 중복도 피하기 어렵습니다.

QASS는 이 과정을 "테스트하면 증적이 따라오는" 구조로 뒤집습니다. 테스터는 평소처럼 페이지를 옮겨 다니며 확인만 하면 되고, 캡처·업로드·정리·1차 결함 점검은 도구가 수행합니다. 검수 담당자는 대시보드에서 팀 전체의 진행 상황을 실시간으로 보면서, 자동 점검이 골라낸 맞춤법·띄어쓰기·UI 장애부터 우선 확인할 수 있습니다.

더 자세한 증적 관리 방법론은 QA 증적 관리 실무 가이드에서, QASS의 구체적인 사용 방법은 사용 가이드에서 확인할 수 있습니다.

자주 묻는 질문

정말 무료인가요?

네. QASS의 모든 기능은 무료로 제공됩니다. 별도의 결제 수단 등록도 필요 없습니다.

회원가입이 필요한가요?

이메일 가입 절차가 없습니다. 이름만 입력하면 바로 입장할 수 있고, 방 단위 비밀번호로 팀 공간을 보호합니다. 같은 방에서 이름이 중복되면 자동으로 안내합니다.

자동 캡처는 어떻게 사용하나요?

크롬 확장 프로그램을 설치한 뒤 방에 연결하고 시작 버튼을 누르면 됩니다. 이후 탭을 이동하거나 스크롤할 때마다 풀페이지 캡처가 방에 자동 업로드됩니다. 자세한 절차는 사용 가이드를 참고하세요.

1차 자동 점검은 무엇을 검사하나요?

캡처 시점의 페이지 텍스트와 화면 요소를 룰 기반으로 검사합니다. 자주 틀리는 맞춤법 (예: "됬습니다" → "됐습니다"), 띄어쓰기 오류(예: "할수있다" → "할 수 있다"), 텍스트 가로 잘림 같은 UI 장애를 검출해 위치와 함께 표시합니다. 원본 이미지는 수정하지 않으며 모든 표시는 화면 위 오버레이로만 이루어집니다.

캡처한 이미지는 어디에 저장되나요?

클라우드 저장소에 방 단위로 저장됩니다. 방을 삭제하면 방에 속한 캡처 이미지도 함께 삭제됩니다. 자세한 내용은 개인정보처리방침을 참고하세요.

모바일에서도 쓸 수 있나요?

대시보드(캡처 열람, 다운로드, 수동 업로드, 자동 점검 결과 확인)는 모바일 브라우저에서 모두 사용할 수 있습니다. 자동 캡처는 크롬 확장 프로그램 기능이므로 PC 크롬에서 동작합니다.
